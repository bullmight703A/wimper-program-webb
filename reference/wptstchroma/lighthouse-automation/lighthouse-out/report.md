## C. Prioritized Backlog

(See 'issues.csv' for full export)
### Recommended Attack Plan
1. **Fix Avoid multiple page redirects**
   - Impact: Systemic (15 pages)
   - Description: Redirects introduce additional delays before the page can be loaded. [Learn how to avoid page redirects](https://developer.chrome.com/docs/lighthouse/performance/redirects/).
2. **Fix Network dependency tree**
   - Impact: Systemic (10 pages)
   - Description: [Avoid chaining critical requests](https://developer.chrome.com/docs/performance/insights/network-dependency-tree) by reducing the length of chains, reducing the download size of resources, or deferring the download of unnecessary resources to improve page load.
3. **Fix Heading elements are not in a sequentially-descending order**
   - Impact: Systemic (8 pages)
   - Description: Properly ordered headings that do not skip levels convey the semantic structure of the page, making it easier to navigate and understand when using assistive technologies. [Learn more about heading order](https://dequeuniversity.com/rules/axe/4.11/heading-order).
4. **Fix Improve image delivery**
   - Impact: Systemic (7 pages)
   - Description: Reducing the download time of images can improve the perceived load time of the page and LCP. [Learn more about optimizing image size](https://developer.chrome.com/docs/performance/insights/image-delivery)
5. **Fix LCP request discovery**
   - Impact: Systemic (5 pages)
   - Description: [Optimize LCP](https://developer.chrome.com/docs/performance/insights/lcp-discovery) by making the LCP image discoverable from the HTML immediately, and avoiding lazy-loading
